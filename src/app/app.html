La projection de contenu n'est pas un input sophistiqué.

Les inputs font descendre les données. Les outputs font monter les événements. La projection de contenu permet au parent de posséder le markup tandis que l'enfant possède la mise en page, le comportement et les limites.

ng-content est un placeholder. Si le parent envoie du contenu, Angular déplace ce contenu dans la vue enfant. Sinon, l'enfant peut conserver un contenu de secours utile.

La projection accepte n'importe quel contenu Angular : texte, bindings, composants, boutons ou formulaires.

Les slots nommés transforment un composant réutilisable en un petit contrat de mise en page. L'enfant expose des slots avec des sélecteurs, puis le parent choisit exactement ce qui va dans chaque slot.

Sélecteurs d'attributs

Les slots peuvent cibler des attributs comme [card-title].

Utilisez des attributs lorsque l'élément projeté n'a pas de classe sémantique propre.

Sélecteurs de classe

Les slots peuvent également cibler des classes comme .card-title.

Utilisez des classes lorsque le contenu appartient déjà à un système de design existant.

<ng-content select="[card-title], .card-title"></ng-content> Un slot peut accepter plusieurs sélecteurs. Cela rend l'API flexible sans ajouter d'inputs.
